# 💾 Database State & System Backup / Restore

Acest document descrie persistența datelor locale și mecanismele de backup/restore din Meridian Onboarding App.

## 📦 Stocare Locală cu `localForage` (IndexedDB)

Deoarece aplicația rulează ca un frontend autonom (fără o bază de date pe server în faza curentă), stocarea datelor se face pe client:
* Se utilizează biblioteca **`localForage`** pentru a interfața cu **IndexedDB**.
* IndexedDB oferă o stocare asincronă robustă, mult mai mare și mai stabilă decât standardul `localStorage`.
* Permite stocarea stării onboarding-ului, a programărilor din scheduler și a profilurilor de angajați adăugați prin panoul de administrare HR.

---

## 📥 Funcționalități de Backup & Restore

Pentru a asigura controlul stării și transferabilitatea datelor între sesiuni de testare sau diferite dispozitive, HR Admin-ul are acces la:
1. **Export System State**: Descarcă întreaga stare a bazei de date sub forma unui fișier JSON structurat.
2. **Import System State**: Permite încărcarea unui fișier JSON exportat anterior pentru a restaura starea bazei de date.

---

## 🔒 Validare cu Schemă Zod

Pentru a evita coruperea bazei de date la importul unui fișier JSON malformat sau incomplet, procesul de restore rulează o validare riguroasă:
* Se definește o schemă **Zod** care validează structura fișierului JSON (câmpuri obligatorii, tipuri de date pentru angajați, structura checklist-ului și programul de scheduler).
* Dacă validarea eșuează, importul este respins cu mesaje clare de eroare.

---

## 👻 Ghost Buddy Audit

La importul stării bazei de date, se realizează un audit special denumit **Ghost Buddy Audit** (Audit Tutore Fantomă):
* **Ce este**: O verificare de consistență referențială.
* **Cum funcționează**: Sistemul parcurge toți angajații și verifică dacă `buddyId` desemnat pentru fiecare angajat indică un utilizator real existent în baza de date importată.
* **Rezultat**: Dacă se găsesc referințe orfane (angajați care au un Buddy ID ce nu există în baza de date), sistemul emite avertismente vizuale clare (Warnings) în panoul de administrare, semnalând administratorului necesitatea de a corecta aceste asocieri.

> [!NOTE]
> Acest audit previne bug-urile de interfață în care profilele de angajați ar încerca să afișeze detalii despre un Buddy inexistent (prevenind crash-uri de tip *undefined*).
